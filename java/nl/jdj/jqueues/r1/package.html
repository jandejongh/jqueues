A small package for simulation of queueing systems.

It uses two important concepts:

  • Jobs (interface SimJob): a job represents an amount of work to be carried out by one or more servers
  • Queues (interface SimQueue): a queue is an abstract waiting line holding jobs to be served by one or more servers associated with the queue
The so-called queue discipline determines in what order jobs are served by the server(s).

An important class of queueing disciplines are the non-preemptive ones, in which jobs, once taken into service, cannot be preempted (and, for instance, later resumed) in favor of another job.
